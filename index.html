About Me

I love waking up and spending my day learning new things and tackling challenging problems. I am currently a Junior Developer in training at Nashville Software School and enjoying every minute. I am passionate about writing better code every day and finding the intersection between software and human needs. I cannot wait to continue my learning journey as a Developer.

My adventure with software development began as a senior in college when one of my friends managed to solve a math problem with a few lines of code. By comparison, I took about 3 hours and 5 pages of writing to solve it. I was immediately intrigued and a spark of interest for Computer Science was ignited. Over the next three years I spent my time and energy teaching high school students mathematics and coaching cross-country, the flame for Computer Science burning in the background.

In the spring of my third year as a teacher I attended a workshop put on by RePublic Schools and Nashville Software School. The workshop taught attendees how to incorporate Computer Science into their lives and classrooms. I was excited to explore a concept that had grabbed my interest many years before but had no idea that this workshop would put me in touch with two organizations that would have such a big impact on my life. Through RePublic, I became the founding high school Computer Science Teacher. Since I was the first person to create and teach this course, I spent countless hours poring over books, articles, and online resources to put together an Introduction to Computer Science curriculum.

When learning the programming portion of the curriculum, I knew that I had found what I wanted to spend my life learning and doing. I applied for the Nashville Software School and it has been the most fun challenge I have ever tackled. Not only have I loved learning as much as possible about the ins and outs of software development, I have also enjoyed getting to know the software development community in Nashville.

Discover America

Helping you plan a trip to a National Park

Technologies Used in this Project: AngularJS, Google Maps, Firebase, Grunt, Bootstrap

This project was the first fully-functioning front-end application that I built. I love to visit National Parks and have been going to at least one every summer for the last four years. Over the years, my friends and I have learned that adequate planning is essential to a safe and fun trip - especially if you want to visit the more remote and adventurous areas of the parks. We found that planning was often difficult because information about the parks are scattered across the Internet in different places. This application was my attempt to centralize information and make it easier to collaboratively plan trips.

    A few lessons learned, on the technical side
  • Do not place asynchronous code inside of synchronous code, weird things will happen
  • Structuring your data is important and the flatter the better when working with NoSQL databases like Firebase.
  • Mobile-first design! I did not do this and it was very difficult to go back and make this application responsive.
  • Get to know the framework you are working in. I did not do as much research on Angular before beginning as I should have. This led to a few brittle solutions that do not support scalability.

The application currently supports planning trips to Yosemite and Glacier National Parks but I would love to expand these offerings!

See it live at Discover America, or on GitHub here.

Users can look at trails and campgrounds in a map or list view

Users can view and edit a daily itinerary of each trip he/she has created

The application uses the dates and the type of trip the user chose to generate the average weather and a suggested packing list

Users can invite other user to collaborate on a trip and notifications will indicate to users when they have an invitation

Explore Nashville

Connecting people with Nashville's art and historic markers.

Technologies Used in this Project: Ionic, Cordova Plugins, AngularJS, Google Maps, Firebase

This project was a group-endeavor built in collaboration with Code for Nashville and the Nashville Historical Commission. Nashville's Chief of Data was at Code for Nashville's Hack Night and proposed the idea for this application. A group of friends and I took on the challenge. It is still a work in progress but we hope to be finished with the first version and have it in the app store by the end of December.

    A few lessons learned
  • It is important, when collaborating, to write detailed user stories and communicate often
  • It can be overwhelming to pick a technology for mobile development since there are many options. Do your research and then do your best to pick the option that meets your needs. We chose Ionic because we wanted to easily offer this application to iOS and Android users.
  • When consuming third-party APIs, make sure you become very familiar with the data that you are receiving before planning out application features.

This application is not currently live but you can view it on GitHub here.

Pinterest Clone

Group Project with Angular

Technologies Used in this Project: AngularJS, Firebase, Grunt, Bootstrap

This project was a group project done at Nashville Software School. The objective was to create a Pinterest Clone using Angular. This was a fun project because we challenged ourselves to experiment with open source Angular libraries and implement difficult features. I learned a lot about Angular and its capabilities during this project.

    A few lessons learned
  • Team-programming will result in a higher quality product than individual programming if you are doing high-quality reviews of code, communicating often, and working toward the same goal.
  • Don't be afraid to experiment! This is a wonderful way to learn.
  • Trying to recreate an application that someone else has built is a great way to push yourself to learn things you may not have thought about learning or doing.

This application is not currently live but you can view it on GitHub here.

  • Category: Front-end and Group Development
